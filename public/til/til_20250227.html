리액트

  • 페이스북에서 개발한 자바스크립트 프레임워크로 사용자 인터페이스를 구축하기 위해 사용된다
  • 단일페이지 에플리케이션(SPA)를 만들때 유용하며 데이터가 변경될 때 효율적으로 UI를 업데이트하는데 강점이 있다

리액트의 정의와 목적

  • 리액트는 컴포넌트 기반 아키텍처를 채택하여 UI를 작은 단위로 나누어 관리한다
  • 주로 프론트엔드 개발에서 사용되며 동적이고 반응형 웹 애플리케이션을 만들기 위해 설계되었다

리액트의 장점

  1. 컴포넌트 기반 : UI를 독립적인 컴포넌트로 나누어 재사용성이 높다
  2. 가상 DOM : 실제 DOM 대신 가상 DOM을 사용해 성능을 최적화한다
  3. 재사용성 : 컴포넌트를 조합해 복잡한 UI를 쉽게 구축할 수 있다
  4. 풍부한 생태계 : 리액트는 다양한 라이브러리와 도구를 지원하며 커뮤니티가 활성화되어있다

리액트를 사용하는 이유

  • SPA(Single Page Application) 지원 : 페이지 전환 없이 동적으로 컨텐츠를 업데이트 할 수 있다
  • 커뮤니티 지원 : 다양한 자료와 튜토리얼이 풍부하다
  • 유연성 : 다른 라이브러리나 프레임워크와 쉽게 통합할 수 있다

JSX(Javascript XML)

  • JSX는 React에서 UI를 구성할 때 사용하는 문법으로 JavaScript를 확장한 문법이다
  • JSX는 브라우저가 실행할 수 있는 JavaScript가 아니므로 코드변환을 해줘야한다
  • Babel이 JSX를 JS로 컴파일하여 브라우저가 읽고 화면에 렌더링 할 수 있다
  • JSX 없이 React 요소를 만들 수 있지만 코드가 복잡하고 가독성이 떨어진다

컴포넌트

  • 함수형 컴포넌트 : 간단한 함수로 정의되며 Hooks를 사용해 상태를 관리한다
  • 클래스형 컴포넌트 : 클래스로 정의되며 생명주기 메서드를 사용할 수 있다
  • 컴포넌트는 독립적이며 여러 컴포넌트를 조합해 복잡한 UI를 만들 수 있다
  • 하나의 기능 구형을 위한 여러 종류의 코드 묶음이다

Props와 State

  • Props
    • Props는 부모 컴포넌트에서 자식 컴포넌트로 데이터를 전달하는데 사용된다
    • props는 키워드가 아니다
  • State
    • 컴포넌트 내부에서 관리되는 상태이다
    • 상태가 변경되면 컴포넌트가 다시 렌더링된다
    • 예시 : const [count, setCount] = useState(0);

이벤트 처리

  • 리액트에서는 이밴트 헨들러를 camelCase로 작성한다(onClick, onChange)
  • 이벤트 핸들러를 통해 State를 업데이트 할 수 있다

DOM과의 차이점

  • React에서는 DOM과 다르게 CSS, JSX 문법만으로 웹 애플리케이션을 개발할 수 있다
  • JSX를 사용하면 JS만으로 마크업 형태의 코드를 작성하여 DOM에 배치할 수 있다
  • JSX는 HTML이 아니기 때문에 Babel을 이용한 컴파일 과정이 필요하다
  • DOM에서 JS와 함꼐 사용하기 위해서 JS와 HTML을 연결하는 작업이 필요하여 Inline 방식이나 Script 태그를 이용하여 외부 JS 파일을 연결할 수 있다
  • React에서는 JSX를 이용하여 앞서 보았던 DOM 코드보다 명시적으로 코드를 작성할 수 있다
  • JS문법과 HTML 문법을 동시에 이용이 가능해 기능과 구조를 한 눈에 확인할 수 있다

JSX 규칙

  • 하나의 엘리먼트 안에서 모든 엘리먼트가 포함
    • JSX에서 여러 엘리먼트를 작성하고자 하는 경우 최상위에서 opening 태그와 closing 태그로 감싸줘야 한다
    • React에서 CSS 속성을 지정하려면 className으로 표기해야 한다
    • 만약 class로 작성하게 되면 React에서는 이를 html클래스 속성 대신 JS 클래스로 받아들이기 때문에 주의
  • JS 표현식 사용 시 중괄호 이용
    • JSX에서 JS를 쓰고자 한다면 중괄호를 이용해야 한다
    • 중괄호를 이용하지 않으면 일반 텍스트로 인식한다
  • 사용자 정의 컴포넌트는 대문자로 시작(PascalCase)
    • React 엘리먼트가 JSX로 작성되면 대문자로 시작해야 한다
    • 이렇게 대문자로 작성된 JSX 컴포넌트를 따로 사용자 정의 컴포넌트라고 부른다
  • 조건부 렌더링에는 삼항연산자 사용
    • 조건부 렌더링은 if문이 아닌 삼항연산자를 이용해야 한다
  • 여러 개의 HTML 엘리먼트를 표시할 때 map() 함수를 이용
    • React에서 여러 개의 HTML 엘리먼트를 표시할 때는 map() 함수를 사용한다
    • map 함수를 사용할 때에는 반드시 key JSX 속성을 넣어야 한다
    • key JSX 속성을 넣지 않으면 리스트의 각 항목에 key를 넣어야 한다는 경고가 표시된다

map을 이용한 반복

  • 배열의 각 요소를 특정 논리(함수)에 의해 다른 요소로 지정(map)한다
  • React에서는 반복적으로 작성해야 하는 부분만 골라서 배열의 요소로 넣으면 React가 이를 인지하고 모든 요소를 렌더링한다
  • map() 함수를 이용할 때는 반드시 “key” 속성을 넣어야 한다
  • key JSX 속성을 넣지 않으면 리스트의 각 항목에 key를 넣어야 한다는 경고가 표시된다
  • return문 안에도 map()을 사용할 수 있다

key 속성

  • React에서 map() 사용 시 key 속성을 넣지 않으면 리스트의 각 항목에 key를 넣어야 한다는 경고가 표시된다
  • key 속성의 위치는 map() 내부에 있는 첫 엘리먼트에 넣어주면 된다
  • key 속성값은 가능하면 데이터에서 제공하는 id를 할당해야 한다
  • key 속성값은 id와 마찬가지로 변하지 않고 예상 가능하며 유일해야 하기 때문이다
  • 고유한 id가 없는 경우에만 배열 인덱스를 넣어서 해결할 수 있지만 배열 인덱스는 최후의 수단으로만 사용한다
