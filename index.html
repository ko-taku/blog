HTML 특징

  • 하이퍼 텍스트 : 태그를 사용하여 웹 페이지 간의 하이퍼링크를 생성
  • 마크업 언어 : 텍스트에 태그를 추가하여 구조와 의미를 정의하는 언어
  • 브라우저에서 해석 : HTML 파일은 웹 브라우저가 읽고 해석하여 사용자에게 웹 페이지로 렌더링

HTML의 기본 구조

  • <!DOCTYPE html>
    • HTML5 문서를 선언하는 코드
    • 브라우저가 HTML5 규격에 맞게 콘텐츠를 해석하도록 지정
  • <html>
    • HTML 문서의 루트 요소로 모든 HTML 태그를 감싼다
  • <head>
    • 문서의 메타 정보를 담는 부분
    • <title> : 브라우저 탭에 표시되는 제목
    • <meta> : 문서 정보(인코딩, 뷰포트 등)를 설정
  • <body>
    • 사용자가 실제로 볼 수 있는 콘텐츠를 담는 부분
    • 텍스트, 이미지, 링크, 폼 등 다양한 요소가 포함

HTML 태그

  • 제목 태그
    • <h1> ~ <h6>
  • 단락 태그
    • <p> : 문장을 단락으로 표시
  • 줄바꿈 태그
    • <br> : 줄을 강제로 바꾼다
  • 수평선 태그
    • <hr> : 문단을 구분하는 선
  • 링크
    • <a href=""> : 다른 페이지나 외부 URL로 이동할 수 있다
  • 이미지
    • <img src="" alt="">
      • Src : 이미지 파일 경로
      • Alt : 이미지를 대체하는 텍스트
  • 목록 태그
    • <ol> : 번호로 구성된 순서가 있는 리스트
      • <li> : 들어가는 항목
    • <ul> : 점으로 구성된 순서 없는 리스트
      • <li> : 들어가는 항목

HTML 속성

  • 속성은 태그에 추가정보를 제공하여 해당 요소의 동작이나 스타일을 설정하는데 사용
  • 속성은 항상 시작 태그 안에서 작성되며 속성명="값"의 형식으로 표현
  • 속성과 값은 공백 없이 등호로 연결되며 값은 큰 따옴표로 묶는 것이 표준

HTML 속성의 주요 특징

  • 모든 태그에 적용 가능
  • 공백 없이 작성
  • 다중 속성 사용 가능

속성의 중요성

  • 요소의 동작 설정
  • CSS 및 Javascript와 연동
    • id와 class 속성은 스타일 지정과 동적 동작 구현의 핵심
  • 접근성과 SEO 향상
    • alt와 title 속성은 웹 접근성과 검색 엔진 최적화에 도움을 준다

자주 사용하는 속성

  • Id: HTML 요소에 고유한 식별자를 부여
  • Class: HTML 요소를 그룹화하여 CSS나 JS에서 스타일 및 동작을 적용
  • Href: 태그에서 링크의 URL을 지정
  • Src: 리소스(이미지, 비디오)의 경로를 지정
  • Alt: 이미지가 로드되지 않을 때 표시되는 텍스트로 접근성을 향상
  • Style: 특정 태그에 CSS 스타일을 직접 적용
  • Title: 요소에 툴팁을 추가
  • 폼 관련 속성
    • Type: 입력 필드의 유형 지정(ex text, password, email)
    • Placeholder: 입력 필드에 힌트를 제공
    • Required: 필수 입력 필드 지정
  • 그 외 HTML5 추가 속성
    • Data-*: 사용자 정의 데이터를 저장하는 커스텀 속성(데이터 처리에 최적화)
    • Controls: 비디오나 오디오 요소에 기본적인 재생 컨트롤을 추가
    • Autoplay: 페이지가 로드되자마자 비디오나 오디오 파일을 자동으로 재생하도록 설정
    • Loop: 비디오나 오디오 파일이 끝까지 재생된 후 자동으로 다시 시작하도록 설정
    • Muted: 소리없이 재생
    • Audio: 사운드 파일 재생
    • Video: 동영상 파일 재생

SEO(Search Engine Optimization)

  • SEO(검색 엔진 최적화): 웹 사이트가 더 높은 순위에 노출하도록 최적화하는 작업
  • 검색 엔진: 특정 키워드에 대해 관련성 높은 콘텐츠를 사용자에게 제공하기 위해 웹 페이지를 색인(index)하고 분석

SEO에 중요한 HTML태그

  1. <title> 태그
    • 페이지의 제목을 정의하며 검색 엔진 결과 페이지(SERP)에 표시
    • 각 페이지마다 고유하고 명확한 제목을 작성
    • 검색 결과에서 역할
      • title>의 내용은 링크로 표시된다
  2. <meta> 태그
    • <meta name="description">
      • 페이지의 요약 정보를 제공하며 검색 결과에서 링크 아래에 표시될 수 있다
    • 검색 결과에서의 역할
      • 이 태그 내용은 검색 결과의 설명으로 표시
    • <meta name="keywords">
      • 페이지와 관련된 키워드를 나열
      • 현재는 검색에서 거의 사용되지 않는다
    • <meta name="robots">
      • 검색 엔진 크롤러에 페이지 크롤링 또는 색인 여부를 지시
    • 옵션
      • Index : 검색 엔진이 페이지를 색인
      • No index : 색인하지 않는다
      • Follow : 링크를 따라감
      • Nofollow : 링크를 따라가지 않는다
  3. 시맨틱 태그(Semantic Tags)
    • 페이지의 구조와 콘텐츠의 의미를 명확히 전달하여 검색 엔진이 페이지를 쉽게 분석할 수 있도록 돕는다
    • 주요 시맨틱 태그
      • <header> : 페이지 또는 섹션의 헤더
      • <nav> : 네비게이션 메뉴
      • <main> : 주요 콘텐츠 영역
      • <article> : 독립적인 콘텐츠
      • <section> : 관련된 콘텐츠 그룹
      • <footer> : 페이지 또는 섹션의 푸터
  4. <h1> ~ <h6> 태그
    • 제목 태그는 콘텐츠의 계층 구조를 나타내며 <h1>은 페이지의 가장 중요한 제목
    • 각 페이지에 하나의 <h1> 태그만 사용하는 것이 좋다
  5. <a> 태그와 앵커 텍스트
    • 링크를 통해 다른 페이지로 연결
    • 앵커 텍스트는 페이지의 주제를 설명하며 검색 엔진이 링크 목적을 이해하는데 도움을 준다
  6. 이미지 태그와 대체 텍스트
    • <img> 태그의 alt 속성은 이미지의 대체 텍스트를 제공하며 이미지가 로드되지 않을 때 표시
    • Alt 텍스트는 검색 엔진이 이미지의 내용을 이해하는데 도움을 준다
  7. URL 구조와 <link> 태그
    • URL
      • 간결하고 키워드를 포함한 URL은 검색 엔진 최적화에 유리
    • Canonical 태그
      • 동일한 콘텐츠가 여러 URL에 존재할 경우 기본 URL을 지정

SEO 태그 사용의 중요성

  1. 검색 엔진에 정보 제공
  2. 검색 순위 향상
  3. 사용자 경험 개선

    • 상태가 success되면 배포한 페이지를 새로고침한다
    • 적용된 것을 확인한다

CSS 기능

  • 콘텐츠의 배치와 위치를 조정하는 레이아웃 디자인
  • 텍스트를 강조하거나 밑줄을 긋는 최소한의 타이포그래피

CSS의 주요 특징

  1. 스타일링의 분리
    • CSS는 HTML에서 스타일을 분리하여 코드의 가독성과 재사용성을 높인다
    • HTML은 구조를 담당하고 CSS는 디자인을 담당하여 역할을 분리
  2. >재사용성
    • 한 번 작성된 CSS 스타일을 여러 HTML 파일에서 재사용할 수 있어 유지보수가 용이
  3. 반응형 디자인
    • CSS는 다양한 화면 크기와 장치에 적응하는 반응형 웹 디자인을 지원
    • 이를 위해 미디어 쿼리와 같은 기술을 활용
  4. 계층적 구조
    • CSS는 스타일 규칙이 계층적으로 적용
    • 스타일의 우선순위를 정의할 수 있다

HTML과 CSS의 연결 방법

  1. 외부 스타일시트 연결
    • 외부 CSS파일을 HTML 문서에 연결할 때 <link> 태그를 사용
    • 구문 : <link rel="stylesheet" href="styles.css">
    • 속성 설명
      • rel="stylesheet": 연결된 파일이 CSS 스타일시트임을 지정한다
      • href="styles.css": 연결할 CSS 파일의 경로를 지정한다
    • 장점
      • 여러 HTML 파일에서 같은 CSS 파일을 재사용 할 수 있다
      • HTML과 CSS가 분리되어 관리가 쉽다
  2. 내부 스타일시트
    • HTML 파일 안에서 <style> 태그를 사용하여 CSS 규칙을 정의할 수 있다
    • 장점
      • 간단한 페이지에서 빠르게 스타일을 적용
    • 단점
      • HTML과 CSS가 섞여 있어 코드 관리가 어렵다
  3. 인라인 스타일
    • HTML 요소에 직접 스타일을 정의할 수 있다
    • 구문 : <h1 style="color: red;">안녕하세요</h1>
    • 장점
      • 특정 요소에 빠르게 스타일을 적용할 수 있다
    • 단점
      • HTML 문서가 복잡해지고, 재사용이 어렵다

외부 스타일시트를 연결할 때 몇 가지 주의해야 할 사항

  1. 경로 설정
    • 상대 경로와 절대 경로
      • href 속성에 스타일시트 파일의 경로를 정확히 지정해야 한다
      • HTML과 CSS 파일이 같은 디렉토리에 있는 경우
        • <link rel="stylesheet" href="styles.css">
      • CSS 파일이 다른 디렉토리에 있을 경우 상대 경로를 올바르게 설정해야 한다
        • <link rel="stylesheet" href="css/styles.css">
      • 절대 경로를 사용할 경우 전체 URL을 지정
        • <link rel="stylesheet" href="https://example.com/styles.css">
    • 경로 오류 방지
      • 파일의 위치를 잘못 설정하면 브라우저가 CSS를 로드할 수 없다
      • 브라우저 개발자 도구(DevTools)를 사용해 파일이 정상적으로 로드되었는지 확인
  2. 파일 이름과 확장자
    • 파일 이름은 명확하게 지정하고, 파일 확장자는 .css여야 한다 (대소문자 주의)
  3. 파일의 인코딩 설정
    • CSS 파일은 UTF-8 인코딩을 사용하는 것이 일반적이다
    • HTML 파일에서도 <meta> 태그로 동일한 인코딩을 지정
      • <meta charset="UTF-8">
  4. <link> 태그 위치
    • <link> 태그는 반드시 <head> 태그 안에 작성
      • <head><link rel="stylesheet" href="styles.css"></head>
    • 주의
      • <body> 태그 내부에 <link>를 넣으면 스타일이 제대로 적용되지 않을 수 있다
  5. 스타일시트의 로딩 순서
    • 여러 CSS 파일을 연결할 경우, 순서대로 로드되므로 우선순위를 고려
    • 첫 번째로 로드된 스타일이 나중에 로드된 스타일에 의해 덮어쓰기(overwrite) 될 수 있다

ID의 특징

  • 유일성
    • ID는 문서 내에서 유일해야 한다
    • 한 요소에만 적용할 수 있으며 특정 요소를 정확히 식별할 때 사용
    • CSS에서는 #id 형식으로 선택

Class의 특징

  • 반복성
    • Class는 여러 요소에 반복적으로 사용할 수 있다
    • 공통 스타일을 적용하려는 경우에 주로 사용
    • CSS에서는 .class 형식으로 선택

ID와 Class를 함께 사용하는 경우

  • ID 스타일이 더 높은 우선순위를 가진다

CSS에서 색상 지정하는 방법

  1. 색상 이름(Color Name)
    • 정의 : CSS에서 미리 정의된 색상 이름을 사용
    • color: red; /* 빨간색 */
    • 장점 : 간단하고 빠르게 색상을 지정
    • 단점 : 사용 가능한 색상이 제한적
  2. 16진수 (Hexadecimal Color)
    • 정의 : #RRGGBB 형식으로 빨강(R), 초록(G), 파랑(B)의 강도를 16진수 값(00~FF)으로 표현
    • color: #ff0000; /* 빨간색 */
    • 장점 : 색상을 정밀하게 지정
    • 단점 : 색상을 직관적으로 이해하기 어렵다
  3. RGB
    • 정의 : rgb(red, green, blue) 형식으로 빨강, 초록, 파랑의 강도를 0~255 범위로 설정
    • color: rgb(255, 0, 0); /* 빨간색 */
    • 장점 : 각 색상의 강도를 세밀하게 조정
  4. RGBA (Red, Green, Blue, Alpha)
    • 정의 : RGB 값에 투명도(Alpha, 0~1)를 추가하여 반투명 색상을 표현
    • background-color: rgba(0, 0, 255, 0.5);
    • 장점 : 색조와 밝기 조절이 직관적
  5. HSL(Hue, Saturation, Lightness)
    • 정의 : 색상(Hue), 채도(Saturation), 명도(Lightness)를 조합하여 색상을 지정
    • color: hsl(120, 100%, 50%); /* 초록색 */
    • 장점 : 색조와 밝기 조절이 직관적

CSS에서 색상을 적용할 수 있는 속성

  • color : 텍스트 색상을 지정
  • background-color : 요소의 배경색을 지정
  • border-color : 테두리의 색상을 지정

CSS 폰트 속성

  1. font-family
    • 글꼴 설정
    • 요소에 사용할 폰트를 지정
    • 여러 폰트를 쉼표로 구분하여 작성
    • 브라우저는 앞에서부터 순서대로 확인하여 사용 가능한 폰트를 선택
    • font-family: 'Arial', 'Helvetica', sans-serif;
  2. font-size
    • 글꼴 크기
    • 텍스트 크기를 설정
    • 단위로는 px, em, %, rem등이 있다
    • font-size: 16px;
  3. font-weight
    • 글꼴 두께
    • 텍스트의 굵기를 설정
    • 값으로는 normal, bold, 또는 숫자(100~900)를 사용할 수 있다
    • font-weight: bold; /* 굵게 */
    • font-weight: 400; /* 기본 두께 */
  4. font-style
    • 글꼴 스타일
    • 텍스트를 이탤릭체 또는 기울임꼴로 만든다
    • font-style: italic;
  5. font-variant
    • 글꼴 변형
    • 소문자 텍스트를 대문자처럼 보이게 하거나 다른 변형 스타일을 적용한다
    • font-variant: small-caps;
  6. font (단축 속성)
    • 글꼴 관련 속성을 한 번에 지정
    • 속성 순서 : font: italic small-caps bold 16px/1.5 'Georgia', serif;

CSS 텍스트 속성

  1. color
    • 텍스트 색상
    • 텍스트의 색상을 설정
    • 값으로 색상 이름, 16진수, RGB, HSL등을 사용
  2. text-align
    • 텍스트 정렬
    • 텍스트의 정렬 위치를 설정
    • 값으로는 left, right, center, justify 등이 있다
    • text-align: center; /* 중앙 정렬 */
  3. line-height
    • 줄 간격
    • 텍스트 줄 간격을 설정
    • 일반적으로 font-size의 배율로 설정하며 가독성을 높이는데 중요
    • line-height: 1.5; /* 1.5배 줄 간격 */
  4. letter-spacing
    • 문자 간격
    • 텍스트 내 각 문자 사이의 간격을 설정
    • 양수 값은 간격을 늘리고, 음수 값은 간격을 좁힌다
    • letter-spacing: 2px; /* 문자 간격 2px 추가 */
  5. text-transform
    • 대소문자 변환
    • 텍스트의 대소문자를 변환
    • 값으로는 uppercase, lowercase, capitalize 등이 있다
    • text-transform: uppercase; /* 모두 대문자로 변환 */
  6. text-decoration
    • 텍스트 장식
    • 텍스트에 밑줄, 취소선, 윗줄 등을 추가
    • text-decoration: underline; /* 밑줄 추가 */
    • text-decoration: line-through; /* 취소선 추가 */

링크 스타일링

  • 기본 링크의 동작 <a>
    • 일반 상태 : 텍스트에 밑줄과 파란색이 기본 적용
    • 호버 상태 : (:hover) 사용자가 마우스를 링크 위로 올렸을 때 스타일이 변경
    • 방문 상태 : (:visited) 사용자가 클릭한 후의 링크에 스타일이 적용
    • 활성 상태 : (:active) 사용자가 링크를 클릭하고 있는 동안의 스타일
  • CSS 속성 설명
    • color : 링크의 텍스트 색상을 설정
    • text-decoration : 링크의 밑줄을 제거하거나 스타일을 추가
    • :hover

버튼 스타일링

  • 기본 버튼의 동작 <button>, <a>
    • background-color : 버튼의 배경색을 지정
    • color : 버튼 텍스트의 색상을 설정
    • padding : 버튼 내부 여백을 설정하여 크기를 조정
    • border : 테두리 스타일을 설정
    • border-radius : 버튼의 모서리를 둥글게 만든다
    • transition : 상태 변화 시 애니메이션 효과를 부드럽게 만든다
    • :hover 마우스를 버튼 위에 올렸을 때 스타일을 지정

Flexbox

  • 1차원 레이아웃 모델
  • 요소를 정렬하고 배치하는데 유연성과 효율성을 제공
  • 행 또는 열을 기준으로 요소를 정렬
  • 컨테이너 안에서 아이템의 크기, 간격, 정렬 방식을 제어

Flexbox의 기본 개념

  • Flex 컨테이너와 Flex 아이템
    • Flex 컨테이너 : Flexbox의 레이아웃을 적용할 부모 요소
    • Flex 아이템 : Flex 컨테이너 안에 포함된 자식 요소
  • 주요 속성
    • 컨테이너 속성
      • display: flex; : flex box 레이아웃을 활성화
      • flex-direction : 아이템 정렬 방향(행 또는 열)을 설정
        • row : 기본값. 왼쪽에서 오른쪽
        • row-reverse : 오른쪽에서 왼쪽
        • column : 위에서 아래로
        • column-reverse : 아래에서 위로
      • justify-content : 주 축(main axis)에서의 정렬 방식
        • flex-start : 왼쪽 정렬
        • flex-end : 오른쪽 정렬
        • center : 가운데 정렬
        • space-between : 아이템 사이에 동일한 간격
        • space-around : 아이템 주변에 동일한 간격
      • align-items : 교차축에서 아이템의 정렬 방식을 설정
        • stretch : 기본값. 아이템의 높이를 동일하게 늘림
        • flex-start : 위쪽 정렬
        • flex-end : 아래쪽 정렬
        • center : 가운데 정렬
        • baseline : 텍스트 기준선에 맞춘다
      • flex-wrap : 아이템이 컨테이너를 초과할 경우 줄바꿈 여부를 설정
        • nowrap : 기본값. 줄바꿈 불가
        • wrap : 줄바꿈 허용
        • wrap-reverse : 줄바꿈을 역방향으로 허용
    • 아이템 속성
      • flex-grow : flex 아이템이 남은 공간을 차지하는 비율
      • flex-shrink : flex 아이템이 줄어드는 비율
      • flex-basis : 아이템의 기본 크기
      • align-self : 특정 아이템의 교차 축 정렬
      • flex : 아이템 공간 분배를 설정

Grid

  • CSS Grid Layout은 웹 페이지에서 2차원 레이아웃(행과 열)을 설계하기 위한 강력한 도구
  • Flex와 달리 Grid는 행과 열을 동시에 제어할 수 있어 복잡한 레이아웃에도 적합

Grid 컨테이너와 Grid 아이템

  • Grid 컨테이너 : Grid 레이아웃이 적용된 부모 요소
  • Grid 아이템 : Grid 컨테이너 안에 포함된 자식 요소
  • 위치와 크기를 설정

컨테이너 속성

  • display : grid; Grid 레이아웃을 활성화
  • grid-template-rows : 행의 크기 정의
  • grid-template-columns : 열의 크기
    • 각 크기는 px, %, fr, auto 등을 사용
  • gap : 행과 열 사이의 간격 설정

아이템 속성

  • grid-column : 아이템이 차지할 열의 범위
  • grid-row : 아이템이 차지할 행의 범위
  • justify-self : 특정 아이템의 수평 정렬
  • align-self : 특정 아이템의 수직 정렬

정렬 속성

  • justify-items : Grid 아이템의 수평 정렬
  • align-items : Grid 아이템의 수직 정렬

flex와 grid의 간단한 차이점

  • flex는 아이템이 우선적으로 크기를 결정
  • grid는 그리드 템플릿 설정이 우선 적용

미디어 쿼리

  • 디바이스의 화면크기, 해상도, 방향에 따라 다른 스타일을 적용할 수 있도록 도와준다
  • 반응형 웹 디자인을 구현할 수 있다

미디어 쿼리 문법

  • max-width
    • 화면의 최대 너비를 기준으로 스타일 적용
    • @media (max-width: 768px) 화면 너비가 768px 이하일 때
  • min-width
    • 화면의 최소 너비를 기준으로 스타일 적용
    • @media (min-width: 1024px) 화면 너비가 1024px 이상일 때
  • min-width and max-width
    • 화면 너비 기준을 내가 정한 기준으로 적용
    • @media (min-width: 500px) and (max-width: 700px)
  • orientation
    • 디바이스의 방향을 기준으로 스타일 적용
    • landscape : 가로 모드
    • portrait : 세로 모드
    • @media (orientation: portrait) 세로 방향일 때
  • resolution
    • 화면 해상도(DPI)를 기준으로 스타일 적용
    • @media (min-resolution: 300dpi)

JavaScript 역사

  • 1955년 탄생
    • 브렌던 아이크에 의해 개발
  • ECMAScript 표준화
    • 1997년에 ECMA 국제 표준화 기구에 의해 표준화
  • 현재의 위치
    • 웹 어플리케이션의 중심 언어
    • React, Angular, Vue.js와 같은 프레임워크와 Node.js를 통해 웹, 서버, 모바일 및 IoT 개발에도 광범위하게 사용

JS 설치

  • JS는 웹브라우저에 내장되어 있어 추가 설치 없이도 실행
  • 서버측 개발이나 로컬 환경에서 실행하기 위해서 Node.js 설치가 필요

Node.js 설치

  1. Node.js 공식 웹사이트에 접속
  2. LTS(Long Term Support) 버전을 다운로드
  3. 설치 후, 터미널에 다음 명령어를 입력하여 설치 여부를 확인
    • node -v
    • nom -v
  4. 설치 후 .js 파일을 생성한 뒤 node 파일이름.js로 실행할 수 있다

JS 특징

  • 인터프리터 언어
    • 코드를 실행 전에 컴파일하지 않고 브라우저가 즉시 해석하여 실행
  • 크로스 플랫폼
    • 브라우저만 있으면 플랫폼에 상관없이 실행 가능
  • 동적 타이핑
    • 변수의 타입을 선언할 필요가 없으며 실행 중에 타입이 동적으로 결정
  • 객체 지향과 함수형 프로그래밍
    • 객체 지향 프로그래밍(OOP)과 함수형 프로그래밍(FP) 모두 지원
  • 이벤트 기반 프로그래밍
    • 사용자 상호작용(클릭, 입력 등)을 처리하기 위해 이벤트 기반 방식으로 동작

JS로 할 수 있는 일

  • 동적인 웹 페이지 생성
    • DOM 조작을 통해 HTML요소를 동적으로 업데이트하거나 CSS를 변경하여 페이지 스타일을 조정
  • 프론트엔드 애플리케이션 개발
    • React, Angular, Vue.js와 같은 프레임워크를 사용해 SPA(Single Page App)를 개발
  • 서버 애플리케이션 개발
    • Node.js를 활용해 서버를 구축하거나 REST API를 개발
  • 모바일 앱 개발
    • React Native, sonic 등을 통해 크로스 플랫폼 모바일 앱을 개발
  • 데스크탑 애플리케이션 개발
    • Electron을 사용해 데스크탑 애플리케이션을 제작
  • 게임 개발
    • Phaser와 같은 게임 엔진을 사용해 2D 또는 3D 게임을 개발
  • 머신러닝 및 데이터 처리
    • TensorFlow.js와 같은 라이브러리를 통해 브라우저에서 머신러닝 작업을 수행

JS에서 주석

  • //다음에 작성
  • /* 과 */ 사이에 작성
  • 주석 처리된 코드는 JS엔진은 인식을 하지 않는다

값과 표현식

  • 표현식
    • 하나의 값으로 평가될 수 있는 코드
  • 값
    • 하나의 고유한 의미를 가지는 코드
  • 연산자
    • 특정 연산을 수행할 수 있는 코드

JS 타입

  • 원시 타입 : 간단한 데이터를 표현하는 타입
    • 1. 숫자
    • 2. 문자열 : 큰따옴표나 작은 따옴표로 감싼다
    • 3. 불리언 : 참 또는 거짓을 나타낸다
    • 4. undefined : 값을 아직 정의하지 않은 상태
    • 5. null : 값이 없음을 명시적으로 표현
    • 6. symbol : 고유한 식별자를 만들 때 사용
  • 타입을 알아야 하는 이유
    • 데이터를 정확히 이해하고 다룰 수 있다
    • 오류를 예방
    • 코드를 깔끔하고 효율적으로 작성 가능
  • 타입 확인 방법
    • typeof 키워드로 확인할 수 있다
  • 원시 타입인 null이 object인 이유
    • 자바스크립트가 처음 설계될 당시 메모리 관리를 위해 데이터 타입을 나타내는 값들이 비트 형태로 저장
    • 객체 타입을 나타내는 내부 비트는 000으로 시작했으며 null의 내부 표현도 000으로 설정
    • 결과적으로 typeof null이 object로 반환
  • Number 타입
    • 정수(integer)와 실수(float)를 모두 표현할 수 있다
    • 같은 숫자 값 간에는 산술 연산자를 사용할 수 있다(+, -, *, /, %)
    • Math 내장 객체
      • 좀 더 복잡한 계산을 위해 사용
      • Math.floor() : 괄호 안의 숫자를 내림하여 반환
      • Math.ceil() : 괄호 안의 숫자를 올림하여 반환
      • Math.round() : 괄호 안의 숫자를 반올림하여 반환
      • Math.abs() : 괄호 안의 숫자의 절대값을 반환
      • Math.sqrt() : 괄호 안의 숫자의 루트값을 반환
      • Math.pow() : 괄호 안의 첫 번째 숫자를 밑, 두 번째 숫자를 지수로 반환
  • String 타입
    • 인간의 언어, 자연어를 JS에서 표현하기 위한 데이터 타입
    • 따옴표, 큰따옴표, 백틱(`)으로 감싸면 된다
    • 맥북에서 백틱은 영어 전환 후 물결(~)을 누르면 입력할 수 있다
    • + 로 문자열을 이어 붙일 수 있다
    • 다른 타입과 이어 붙이려고 하면 모두 문자열로 변하므로 조심해서 사용해야 한다
    • 문자열의 length 속성을 이용하여 문자열의 길이를 확인할 수 있다
    • 문자열 값에 .length를 붙이면 된다
    • 각 문자가 몇 번째에 위치하는지 인덱스로 확인할 수 있다
    • 문자열 주요 메서드
      • toLowerCase() : 문자열을 소문자로 변경
      • toUpperCase() : 문자열을 대문자로 변경
      • concat() : 문자열 연결 연산자 + 처럼 문자열을 이어붙일 수 있다
      • slice() : 문자열의 일부를 자를 수 있다
      • indexOf() : 문자열 내에 특정 문자나 숫자가 몇 번째 위치하는지 확인한다
      • includes() : 문자열 내에 특정 문자가 포함되어 있는지 확인한다
  • Boolean 타입
    • 사실관계를 구분하기 위한 타입
    • true 혹은 false 둘 중 하나의 값
    • falsy : 불리언 타입은 아니지만 JS로 false로 여겨지는 값
      • 0
      • -0
      • 0n
      • ""
      • ''
      • ``
      • null
      • undefined
      • NaN
    • 불리언은 비교연산자로 두 값이 같은지 다른지를 확인할 때 유용하다
    • 엄격한 동치 연산자
      • ===, !==
      • 두 피연산자 값과 타입이 같으면 true, 다르면 false를 반환
    • 느슨한 동치 연산자
      • ==, !=
      • 예외가 많아 사용을 권장하지 않는다
      • 숫자열과 문자열의 값이 같아도 true로 반환한다
    • 대소 관계 비교 연산자
      • >, <,>=, <=< /li>
      • 두 피연산자의 값의 크기를 비교
    • 논리연산자
      • 두 값 간의 논리 관계를 확인
      • || : 논리합(or)
        • 두 값 중 하나만 true여도 true로 판단
        • 두 값이 모두 false면 false로 판단
      • && : 논리곱(AND)
        • 두 값이 모두 true면 true로 판단
        • 두 값 중 하나만 false여도 false로 판단
    • 논리 부정 연산자
      • 사실 관계를 반대로 표현
      • ! : 부정(not)
        • 오른쪽 피연산자와 반대의 사실을 반환
        • false, truthy의 반대 값을 반환

JS변수 선언

  • 아무것도 할당하지 않은 변수는 undefined가 자동으로 할당
  • let
    • let은 블록 스코프를 가진다
    • 코드 블록 내에서 선언된 변수는 코드 블록 내에서만 유효
    • 초기화되지 않은 상태에서 호이스팅으로 접근하면 TDZ(Terminal Dead Zone) 오류
    • 동일 스코프 내 같은 이름으로 재선언 불가
    • 값 변경 가능
  • const
    • const는 블록 스코프를 가진다
    • 코드 블록 내에서 선언된 변수는 코드 블록 내에서만 유효
    • 초기화되지 않은 상태에서 호이스팅으로 접근하면 TDZ 오류
    • 동일 스코프 내에서 같은 이름으로 재선언 불가
    • const는 상수로서 값을 한 번만 할당
    • 객체나 배열같은 참조형 데이터 타입의 경우 내부 값 변경 가능
  • 네이밍 규칙
    • 식별자는 특수문자를 제외한 문자, 언더스코어(_), 달러 기호($)로 시작해야 한다
    • 식별자는 특수문자를 제외한 문자, 숫자, 언더스코어, 달러 기호를 포함할 수 있다
    • 예약어는 식별자로 사용할 수 없다
      • 예약어 : 프로그래밍 언어에서 사용되고 있거나 사용될 예정인 단어(let, const, true 등)
  • 네이밍 컨벤션
    • 카멜케이스
    • 첫 단어의 첫 문자는 소문자로 그 이후에 연결되는 단어의 첫 문자는 대문자로 작성
    • 예 : lastName

변수 활용

  • 변수에 숫자 타입이 할당되어 있는 경우 숫자 타입에 사용 가능한 모든 연산이 가능
  • 연산 결과를 변수에 반영하려면 재할당 필요
  • 변수끼리의 연산도 가능

함수 정의

  • 입력을 받아서 코드블록 내부의 코드를 실행한 후 함수의 실행 결과를 반환하는 일련의 과정 묶음
    • 1. 함수 외부에서 특정한 입력값을 전달하여 호출
    • 2. 함수 내부의 코드가 실행
    • 3. 함수의 실행 결과는 함수 외부로 반환(Return)

함수선언문과 함수표현식

  • 함수선언문
    • function 키워드 사용
    • 키워드 다음으로 함수명을 지정
    • 소괄호에서 함수 내부에 사용하는 매개변수를 사용
    • 중괄호 안에 실행할 코드 블록을 넣어준다
  • 함수표현식
    • let 키워드를 사용해서 변수를 선언
    • 함수를 할당하는 형태로 코드를 작성
    • 함수에는 특별한 식별자를 지정할 필요가 없다
  • 함수 호출
    • 함수를 정의할 때 지정한 함수명 뒤에 소괄호를 붙이면 함수를 호출할 수 있다
    • 함수표현식은 선언한 변수명에 소괄호를 붙이면 된다

매개변수와 전달인자

  • 매개변수
    • 매개변수는 함수를 정의할 때 선언
    • 함수 코드 블록 안에서 변수처럼 취급
    • 함수를 정의할 때 소괄호 안에 매개변수 추가할 수 있다
    • 여러 개의 매개변수를 사용할 수 있다
  • 전달인자
    • 함수를 호출할 때 소괄호 안에 값을 넣는다
    • 값을 넣음으로서 매개변수에 값을 할당
    • 매개변수의 개수에 맞게 전달인자를 전달
    • 매개변수보다 적은 전달인자가 전달되면 전달되지 않은 매개변수는 undefined로 초기화
  • 스코프
    • 변수가 유효한 범위
    • 매개변수는 함수 내부에서만 사용이 가능
    • 함수 내부에서 선언한 변수도 함수 내부에서만 사용 가능

return문

  • return
    • 함수의 실행결과를 외부로 반환
    • 함수 내부의 코드가 차례대로 실행되다가 return문을 만나면 값을 반환한 후 함수는 종료
    • return문 뒤에 나오는 코드는 실행되지 않는다
    • return문에 작성된 코드를 실행한 후 결과를 함수 외부로 리턴
    • 함수 외부에서 함수를 호출하면 함수 실행결과를 확인할 수 있다
    • 함수 호출의 결과를 변수에 할당하는 것도 가능
    • 함수의 호출 결과끼리의 연산도 가능

타입스크립트

  • JS의 한계를 극복하기 위해 MS사가 정적 타입 문법을 추가한 프로그래밍 언어

JS의 한계점

  • 동적 타입 언어 특성
    • 자바스크립트(JS)는 동적 타입 언어로, 변수의 타입이 런타임에 결정
    • 이로 인해 변수의 타입을 명시적으로 정의하지 않아, 여러 상황에서 예기치 않은 동작이나 오류가 발생할 수 있다
    • 타입 관련 오류 : JS는 타입이 유연하다는 장점이 있지만, 이로 인해 잘못된 타입을 변수에 할당하는 실수를 할 확률이 높다
    • 예를 들어, undefined나 null 값이 변수에 들어가면 의도치 않은 결과가 발생할 수 있다
    • 타입 추론 : JS는 변수 선언 시 타입을 명시하지 않기 때문에, 자동으로 타입을 추론하게 된다
    • 이로 인해 타입이 잘못 추론되거나 의도치 않은 자동 타입 변환이 발생할 수 있다
  • 객체지향 프로그래밍 지원 부족
    • 클래스는 ES6에서 도입되었지만, JS는 본래 프로토타입 기반 객체 지향 언어
    • ES6 이전에는 클래스를 사용하지 않고 객체를 구성하려면 복잡한 방법을 사용
    • JS는 클래스 기반 객체 지향 언어에서 제공하는 많은 기능을 제공하지 않는다
    • 대규모 애플리케이션에서 상속이나 인터페이스와 같은 OOP 특성의 활용에 제한이 있을 수 있다
  • 코드 가독성 및 유지보수성
    • JS는 변수 타입을 명시하지 않기 때문에 가독성이 떨어질 수 있다
    • 코드가 길어지면 변수와 함수가 어떤 역할을 하는지 명확히 알기 어렵다
  • 대규모 프로젝트
    • 코드베이스가 커지거나 복잡해질수록 타입 정의가 중요한 역할을 한다
    • JS는 타입 시스템이 약하고 코드가 길어질수록 타입 관련 추적이 어려워져 디버깅이나 리팩토링이 힘들어진다
    • 다양한 브라우저 환경에서도 예상치 못한 호환성 문제를 유발할 수 있다
  • 성능 문제
    • JS는 런타임에서 동적으로 타입을 확인하고 처리
    • 성능이 중요한 곳에서는 정적 타입 언어가 더 유리
  • 병렬 처리
    • JS는 싱글 스레드 모델로 동작
    • 멀티스레딩이 필요한 환경에서 한계가 있을 수 있다
    • 이를 해결하기 위해 웹 워커와 같은 방법을 사용하지만 여전히 한계가 존재

타입스크립트의 주요 특징

  1. 정적 타입 시스템
    • 타입스크립트는 정적 타입을 제공
    • 코드 작성 시 변수의 타입을 미리 정의
    • 컴파일 시 타임 오류를 발견할 수 있기 때문에 런타임에서 발생할 수 있는 오류를 미리 방지
    • 타입을 명시함으로서 타입 오류를 방지하고 코드의 안정성을 높일 수 있다
  2. 객체 지향 프로그래밍(OOP)
    • 타입스크립트는 자바처럼 클래스 및 인터페이스 기반의 객체 지향 프로그래밍을 지원
    • JS의 프로토타입 기반 객체 모델을 보완하며 더 직관적이고 강력한 객체 지향 방식을 제공
  3. JS와 완전 호환
    • 타입스크립트는 JS의 슈퍼셋
    • 즉 JS코드는 그대로 타입스크립트에서 실행할 수 있다
  4. ES6+ 지원
    • 타입스크립트는 최신 자바스크립트 기능을 지원
    • 모듈 시스템, 비동기 처리, 어싱크/어웨이트 등의 기능도 사용
    • 트랜스파일러를 사용하여 이전 버전의 JS로 변환 가능

타입스크립트 IDE 및 툴

  • Visual Studio Code (VS Code)
    • VS Code는 타입스크립트 개발을 위한 가장 인기 있는 IDE 중 하나이다
    • 타입스크립트를 기본적으로 지원하며, 자동 완성, 타입 오류 표시, 디버깅 등 다양한 기능을 제공한다
    • 타입스크립트 서버를 내장하여, 실시간으로 타입 오류를 확인하고 빠르게 수정할 수 있다
  • WebStorm
    • WebStorm은 JetBrains에서 제공하는 강력한 웹 개발 IDE로, 타입스크립트를 지원한다
    • 뛰어난 코드 자동 완성 및 타입 검증 기능을 제공하여, 큰 프로젝트에서도 효율적으로 작업할 수 있다
  • TypeScript Playground
    • TypeScript Playground는 브라우저에서 바로 타입스크립트 코드를 작성하고 실행할 수 있는 온라인 도구이다
    • 타입스크립트의 문법을 익히거나 실험적인 코드를 테스트할 때 유용하다
  • TSlint
    • TSlint는 타입스크립트 코드에서 코딩 규칙을 정의하고 강제할 수 있는 도구이다
    • TSLint를 사용하면 팀 내에서 일정한 스타일을 유지하며 개발할 수 있다
  • ESLint
    • ESLint는 자바스크립트 및 타입스크립트 코드에서 문법 오류 및 스타일 오류를 검사할 수 있는 도구로, 타입스크립트용으로 설정할 수 있다
    • TSLint는 더 이상 활발히 유지보수되지 않으므로, ESLint를 대신 사용한다

타입스크립트와 호환되는 라이브러리 및 프레임워크

  • React
    • 타입스크립트는 React와 완벽하게 호환된다
    • React에서 타입스크립트를 사용할 때, JSX의 타입을 명시하거나 Props, State의 타입을 선언함으로써 컴파일 타임에 오류를 미리 잡을 수 있다
    • @types/react와 같은 타입 정의 파일을 설치하여 React 프로젝트에서 타입스크립트를 사용할 수 있다
  • Vue
    • Vue 3.x부터는 타입스크립트를 첫 번째 클래스 언어로 지원하며, Vue의 API가 타입스크립트를 사용하기에 최적화되어 있다
    • Vue 컴포넌트에서 타입스크립트를 사용하여 더욱 안전하고 유지보수성 높은 코드를 작성할 수 있다
  • Node.js
    • Node.js에서도 타입스크립트를 사용할 수 있다
    • Express와 같은 웹 서버 프레임워크에서 타입스크립트를 사용하여 더 안전하고 읽기 쉬운 코드를 작성할 수 있다
    • 또한, @types/express와 같은 타입 정의 파일을 사용하면, Express 코드에서 발생할 수 있는 오류를 미리 방지할 수 있다
  • Angular
    • Angular는 타입스크립트 기반으로 개발된 프레임워크로, 타입스크립트의 주요 장점을 그대로 활용할 수 있다
    • Angular CLI는 기본적으로 타입스크립트를 지원하며, 컴포넌트, 서비스, 모듈의 타입을 명시적으로 선언할 수 있다
  • D3.js
    • D3.js는 데이터 시각화 라이브러리로, 타입스크립트와 함께 사용하면 타입 안정성을 유지하면서 강력한 시각화를 구현할 수 있다
    • @types/d3와 같은 타입 정의 파일을 통해 D3의 타입을 정의하여 더 안전한 코드를 작성할 수 있다
