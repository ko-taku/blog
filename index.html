HTML 특징

  • 하이퍼 텍스트 : 태그를 사용하여 웹 페이지 간의 하이퍼링크를 생성
  • 마크업 언어 : 텍스트에 태그를 추가하여 구조와 의미를 정의하는 언어
  • 브라우저에서 해석 : HTML 파일은 웹 브라우저가 읽고 해석하여 사용자에게 웹 페이지로 렌더링

HTML의 기본 구조

  • <!DOCTYPE html>
    • HTML5 문서를 선언하는 코드
    • 브라우저가 HTML5 규격에 맞게 콘텐츠를 해석하도록 지정
  • <html>
    • HTML 문서의 루트 요소로 모든 HTML 태그를 감싼다
  • <head>
    • 문서의 메타 정보를 담는 부분
    • <title> : 브라우저 탭에 표시되는 제목
    • <meta> : 문서 정보(인코딩, 뷰포트 등)를 설정
  • <body>
    • 사용자가 실제로 볼 수 있는 콘텐츠를 담는 부분
    • 텍스트, 이미지, 링크, 폼 등 다양한 요소가 포함

HTML 태그

  • 제목 태그
    • <h1> ~ <h6>
  • 단락 태그
    • <p> : 문장을 단락으로 표시
  • 줄바꿈 태그
    • <br> : 줄을 강제로 바꾼다
  • 수평선 태그
    • <hr> : 문단을 구분하는 선
  • 링크
    • <a href=""> : 다른 페이지나 외부 URL로 이동할 수 있다
  • 이미지
    • <img src="" alt="">
      • Src : 이미지 파일 경로
      • Alt : 이미지를 대체하는 텍스트
  • 목록 태그
    • <ol> : 번호로 구성된 순서가 있는 리스트
      • <li> : 들어가는 항목
    • <ul> : 점으로 구성된 순서 없는 리스트
      • <li> : 들어가는 항목

HTML 속성

  • 속성은 태그에 추가정보를 제공하여 해당 요소의 동작이나 스타일을 설정하는데 사용
  • 속성은 항상 시작 태그 안에서 작성되며 속성명="값"의 형식으로 표현
  • 속성과 값은 공백 없이 등호로 연결되며 값은 큰 따옴표로 묶는 것이 표준

HTML 속성의 주요 특징

  • 모든 태그에 적용 가능
  • 공백 없이 작성
  • 다중 속성 사용 가능

속성의 중요성

  • 요소의 동작 설정
  • CSS 및 Javascript와 연동
    • id와 class 속성은 스타일 지정과 동적 동작 구현의 핵심
  • 접근성과 SEO 향상
    • alt와 title 속성은 웹 접근성과 검색 엔진 최적화에 도움을 준다

자주 사용하는 속성

  • Id: HTML 요소에 고유한 식별자를 부여
  • Class: HTML 요소를 그룹화하여 CSS나 JS에서 스타일 및 동작을 적용
  • Href: 태그에서 링크의 URL을 지정
  • Src: 리소스(이미지, 비디오)의 경로를 지정
  • Alt: 이미지가 로드되지 않을 때 표시되는 텍스트로 접근성을 향상
  • Style: 특정 태그에 CSS 스타일을 직접 적용
  • Title: 요소에 툴팁을 추가
  • 폼 관련 속성
    • Type: 입력 필드의 유형 지정(ex text, password, email)
    • Placeholder: 입력 필드에 힌트를 제공
    • Required: 필수 입력 필드 지정
  • 그 외 HTML5 추가 속성
    • Data-*: 사용자 정의 데이터를 저장하는 커스텀 속성(데이터 처리에 최적화)
    • Controls: 비디오나 오디오 요소에 기본적인 재생 컨트롤을 추가
    • Autoplay: 페이지가 로드되자마자 비디오나 오디오 파일을 자동으로 재생하도록 설정
    • Loop: 비디오나 오디오 파일이 끝까지 재생된 후 자동으로 다시 시작하도록 설정
    • Muted: 소리없이 재생
    • Audio: 사운드 파일 재생
    • Video: 동영상 파일 재생

SEO(Search Engine Optimization)

  • SEO(검색 엔진 최적화): 웹 사이트가 더 높은 순위에 노출하도록 최적화하는 작업
  • 검색 엔진: 특정 키워드에 대해 관련성 높은 콘텐츠를 사용자에게 제공하기 위해 웹 페이지를 색인(index)하고 분석

SEO에 중요한 HTML태그

  1. <title> 태그
    • 페이지의 제목을 정의하며 검색 엔진 결과 페이지(SERP)에 표시
    • 각 페이지마다 고유하고 명확한 제목을 작성
    • 검색 결과에서 역할
      • title>의 내용은 링크로 표시된다
  2. <meta> 태그
    • <meta name="description">
      • 페이지의 요약 정보를 제공하며 검색 결과에서 링크 아래에 표시될 수 있다
    • 검색 결과에서의 역할
      • 이 태그 내용은 검색 결과의 설명으로 표시
    • <meta name="keywords">
      • 페이지와 관련된 키워드를 나열
      • 현재는 검색에서 거의 사용되지 않는다
    • <meta name="robots">
      • 검색 엔진 크롤러에 페이지 크롤링 또는 색인 여부를 지시
    • 옵션
      • Index : 검색 엔진이 페이지를 색인
      • No index : 색인하지 않는다
      • Follow : 링크를 따라감
      • Nofollow : 링크를 따라가지 않는다
  3. 시맨틱 태그(Semantic Tags)
    • 페이지의 구조와 콘텐츠의 의미를 명확히 전달하여 검색 엔진이 페이지를 쉽게 분석할 수 있도록 돕는다
    • 주요 시맨틱 태그
      • <header> : 페이지 또는 섹션의 헤더
      • <nav> : 네비게이션 메뉴
      • <main> : 주요 콘텐츠 영역
      • <article> : 독립적인 콘텐츠
      • <section> : 관련된 콘텐츠 그룹
      • <footer> : 페이지 또는 섹션의 푸터
  4. <h1> ~ <h6> 태그
    • 제목 태그는 콘텐츠의 계층 구조를 나타내며 <h1>은 페이지의 가장 중요한 제목
    • 각 페이지에 하나의 <h1> 태그만 사용하는 것이 좋다
  5. <a> 태그와 앵커 텍스트
    • 링크를 통해 다른 페이지로 연결
    • 앵커 텍스트는 페이지의 주제를 설명하며 검색 엔진이 링크 목적을 이해하는데 도움을 준다
  6. 이미지 태그와 대체 텍스트
    • <img> 태그의 alt 속성은 이미지의 대체 텍스트를 제공하며 이미지가 로드되지 않을 때 표시
    • Alt 텍스트는 검색 엔진이 이미지의 내용을 이해하는데 도움을 준다
  7. URL 구조와 <link> 태그
    • URL
      • 간결하고 키워드를 포함한 URL은 검색 엔진 최적화에 유리
    • Canonical 태그
      • 동일한 콘텐츠가 여러 URL에 존재할 경우 기본 URL을 지정

SEO 태그 사용의 중요성

  1. 검색 엔진에 정보 제공
  2. 검색 순위 향상
  3. 사용자 경험 개선

    • 상태가 success되면 배포한 페이지를 새로고침한다
    • 적용된 것을 확인한다

CSS 기능

  • 콘텐츠의 배치와 위치를 조정하는 레이아웃 디자인
  • 텍스트를 강조하거나 밑줄을 긋는 최소한의 타이포그래피

CSS의 주요 특징

  1. 스타일링의 분리
    • CSS는 HTML에서 스타일을 분리하여 코드의 가독성과 재사용성을 높인다
    • HTML은 구조를 담당하고 CSS는 디자인을 담당하여 역할을 분리
  2. >재사용성
    • 한 번 작성된 CSS 스타일을 여러 HTML 파일에서 재사용할 수 있어 유지보수가 용이
  3. 반응형 디자인
    • CSS는 다양한 화면 크기와 장치에 적응하는 반응형 웹 디자인을 지원
    • 이를 위해 미디어 쿼리와 같은 기술을 활용
  4. 계층적 구조
    • CSS는 스타일 규칙이 계층적으로 적용
    • 스타일의 우선순위를 정의할 수 있다

HTML과 CSS의 연결 방법

  1. 외부 스타일시트 연결
    • 외부 CSS파일을 HTML 문서에 연결할 때 <link> 태그를 사용
    • 구문 : <link rel="stylesheet" href="styles.css">
    • 속성 설명
      • rel="stylesheet": 연결된 파일이 CSS 스타일시트임을 지정한다
      • href="styles.css": 연결할 CSS 파일의 경로를 지정한다
    • 장점
      • 여러 HTML 파일에서 같은 CSS 파일을 재사용 할 수 있다
      • HTML과 CSS가 분리되어 관리가 쉽다
  2. 내부 스타일시트
    • HTML 파일 안에서 <style> 태그를 사용하여 CSS 규칙을 정의할 수 있다
    • 장점
      • 간단한 페이지에서 빠르게 스타일을 적용
    • 단점
      • HTML과 CSS가 섞여 있어 코드 관리가 어렵다
  3. 인라인 스타일
    • HTML 요소에 직접 스타일을 정의할 수 있다
    • 구문 : <h1 style="color: red;">안녕하세요</h1>
    • 장점
      • 특정 요소에 빠르게 스타일을 적용할 수 있다
    • 단점
      • HTML 문서가 복잡해지고, 재사용이 어렵다

외부 스타일시트를 연결할 때 몇 가지 주의해야 할 사항

  1. 경로 설정
    • 상대 경로와 절대 경로
      • href 속성에 스타일시트 파일의 경로를 정확히 지정해야 한다
      • HTML과 CSS 파일이 같은 디렉토리에 있는 경우
        • <link rel="stylesheet" href="styles.css">
      • CSS 파일이 다른 디렉토리에 있을 경우 상대 경로를 올바르게 설정해야 한다
        • <link rel="stylesheet" href="css/styles.css">
      • 절대 경로를 사용할 경우 전체 URL을 지정
        • <link rel="stylesheet" href="https://example.com/styles.css">
    • 경로 오류 방지
      • 파일의 위치를 잘못 설정하면 브라우저가 CSS를 로드할 수 없다
      • 브라우저 개발자 도구(DevTools)를 사용해 파일이 정상적으로 로드되었는지 확인
  2. 파일 이름과 확장자
    • 파일 이름은 명확하게 지정하고, 파일 확장자는 .css여야 한다 (대소문자 주의)
  3. 파일의 인코딩 설정
    • CSS 파일은 UTF-8 인코딩을 사용하는 것이 일반적이다
    • HTML 파일에서도 <meta> 태그로 동일한 인코딩을 지정
      • <meta charset="UTF-8">
  4. <link> 태그 위치
    • <link> 태그는 반드시 <head> 태그 안에 작성
      • <head><link rel="stylesheet" href="styles.css"></head>
    • 주의
      • <body> 태그 내부에 <link>를 넣으면 스타일이 제대로 적용되지 않을 수 있다
  5. 스타일시트의 로딩 순서
    • 여러 CSS 파일을 연결할 경우, 순서대로 로드되므로 우선순위를 고려
    • 첫 번째로 로드된 스타일이 나중에 로드된 스타일에 의해 덮어쓰기(overwrite) 될 수 있다

ID의 특징

  • 유일성
    • ID는 문서 내에서 유일해야 한다
    • 한 요소에만 적용할 수 있으며 특정 요소를 정확히 식별할 때 사용
    • CSS에서는 #id 형식으로 선택

Class의 특징

  • 반복성
    • Class는 여러 요소에 반복적으로 사용할 수 있다
    • 공통 스타일을 적용하려는 경우에 주로 사용
    • CSS에서는 .class 형식으로 선택

ID와 Class를 함께 사용하는 경우

  • ID 스타일이 더 높은 우선순위를 가진다

CSS에서 색상 지정하는 방법

  1. 색상 이름(Color Name)
    • 정의 : CSS에서 미리 정의된 색상 이름을 사용
    • color: red; /* 빨간색 */
    • 장점 : 간단하고 빠르게 색상을 지정
    • 단점 : 사용 가능한 색상이 제한적
  2. 16진수 (Hexadecimal Color)
    • 정의 : #RRGGBB 형식으로 빨강(R), 초록(G), 파랑(B)의 강도를 16진수 값(00~FF)으로 표현
    • color: #ff0000; /* 빨간색 */
    • 장점 : 색상을 정밀하게 지정
    • 단점 : 색상을 직관적으로 이해하기 어렵다
  3. RGB
    • 정의 : rgb(red, green, blue) 형식으로 빨강, 초록, 파랑의 강도를 0~255 범위로 설정
    • color: rgb(255, 0, 0); /* 빨간색 */
    • 장점 : 각 색상의 강도를 세밀하게 조정
  4. RGBA (Red, Green, Blue, Alpha)
    • 정의 : RGB 값에 투명도(Alpha, 0~1)를 추가하여 반투명 색상을 표현
    • background-color: rgba(0, 0, 255, 0.5);
    • 장점 : 색조와 밝기 조절이 직관적
  5. HSL(Hue, Saturation, Lightness)
    • 정의 : 색상(Hue), 채도(Saturation), 명도(Lightness)를 조합하여 색상을 지정
    • color: hsl(120, 100%, 50%); /* 초록색 */
    • 장점 : 색조와 밝기 조절이 직관적

CSS에서 색상을 적용할 수 있는 속성

  • color : 텍스트 색상을 지정
  • background-color : 요소의 배경색을 지정
  • border-color : 테두리의 색상을 지정

CSS 폰트 속성

  1. font-family
    • 글꼴 설정
    • 요소에 사용할 폰트를 지정
    • 여러 폰트를 쉼표로 구분하여 작성
    • 브라우저는 앞에서부터 순서대로 확인하여 사용 가능한 폰트를 선택
    • font-family: 'Arial', 'Helvetica', sans-serif;
  2. font-size
    • 글꼴 크기
    • 텍스트 크기를 설정
    • 단위로는 px, em, %, rem등이 있다
    • font-size: 16px;
  3. font-weight
    • 글꼴 두께
    • 텍스트의 굵기를 설정
    • 값으로는 normal, bold, 또는 숫자(100~900)를 사용할 수 있다
    • font-weight: bold; /* 굵게 */
    • font-weight: 400; /* 기본 두께 */
  4. font-style
    • 글꼴 스타일
    • 텍스트를 이탤릭체 또는 기울임꼴로 만든다
    • font-style: italic;
  5. font-variant
    • 글꼴 변형
    • 소문자 텍스트를 대문자처럼 보이게 하거나 다른 변형 스타일을 적용한다
    • font-variant: small-caps;
  6. font (단축 속성)
    • 글꼴 관련 속성을 한 번에 지정
    • 속성 순서 : font: italic small-caps bold 16px/1.5 'Georgia', serif;

CSS 텍스트 속성

  1. color
    • 텍스트 색상
    • 텍스트의 색상을 설정
    • 값으로 색상 이름, 16진수, RGB, HSL등을 사용
  2. text-align
    • 텍스트 정렬
    • 텍스트의 정렬 위치를 설정
    • 값으로는 left, right, center, justify 등이 있다
    • text-align: center; /* 중앙 정렬 */
  3. line-height
    • 줄 간격
    • 텍스트 줄 간격을 설정
    • 일반적으로 font-size의 배율로 설정하며 가독성을 높이는데 중요
    • line-height: 1.5; /* 1.5배 줄 간격 */
  4. letter-spacing
    • 문자 간격
    • 텍스트 내 각 문자 사이의 간격을 설정
    • 양수 값은 간격을 늘리고, 음수 값은 간격을 좁힌다
    • letter-spacing: 2px; /* 문자 간격 2px 추가 */
  5. text-transform
    • 대소문자 변환
    • 텍스트의 대소문자를 변환
    • 값으로는 uppercase, lowercase, capitalize 등이 있다
    • text-transform: uppercase; /* 모두 대문자로 변환 */
  6. text-decoration
    • 텍스트 장식
    • 텍스트에 밑줄, 취소선, 윗줄 등을 추가
    • text-decoration: underline; /* 밑줄 추가 */
    • text-decoration: line-through; /* 취소선 추가 */

링크 스타일링

  • 기본 링크의 동작 <a>
    • 일반 상태 : 텍스트에 밑줄과 파란색이 기본 적용
    • 호버 상태 : (:hover) 사용자가 마우스를 링크 위로 올렸을 때 스타일이 변경
    • 방문 상태 : (:visited) 사용자가 클릭한 후의 링크에 스타일이 적용
    • 활성 상태 : (:active) 사용자가 링크를 클릭하고 있는 동안의 스타일
  • CSS 속성 설명
    • color : 링크의 텍스트 색상을 설정
    • text-decoration : 링크의 밑줄을 제거하거나 스타일을 추가
    • :hover

버튼 스타일링

  • 기본 버튼의 동작 <button>, <a>
    • background-color : 버튼의 배경색을 지정
    • color : 버튼 텍스트의 색상을 설정
    • padding : 버튼 내부 여백을 설정하여 크기를 조정
    • border : 테두리 스타일을 설정
    • border-radius : 버튼의 모서리를 둥글게 만든다
    • transition : 상태 변화 시 애니메이션 효과를 부드럽게 만든다
    • :hover 마우스를 버튼 위에 올렸을 때 스타일을 지정

Flexbox

  • 1차원 레이아웃 모델
  • 요소를 정렬하고 배치하는데 유연성과 효율성을 제공
  • 행 또는 열을 기준으로 요소를 정렬
  • 컨테이너 안에서 아이템의 크기, 간격, 정렬 방식을 제어

Flexbox의 기본 개념

  • Flex 컨테이너와 Flex 아이템
    • Flex 컨테이너 : Flexbox의 레이아웃을 적용할 부모 요소
    • Flex 아이템 : Flex 컨테이너 안에 포함된 자식 요소
  • 주요 속성
    • 컨테이너 속성
      • display: flex; : flex box 레이아웃을 활성화
      • flex-direction : 아이템 정렬 방향(행 또는 열)을 설정
        • row : 기본값. 왼쪽에서 오른쪽
        • row-reverse : 오른쪽에서 왼쪽
        • column : 위에서 아래로
        • column-reverse : 아래에서 위로
      • justify-content : 주 축(main axis)에서의 정렬 방식
        • flex-start : 왼쪽 정렬
        • flex-end : 오른쪽 정렬
        • center : 가운데 정렬
        • space-between : 아이템 사이에 동일한 간격
        • space-around : 아이템 주변에 동일한 간격
      • align-items : 교차축에서 아이템의 정렬 방식을 설정
        • stretch : 기본값. 아이템의 높이를 동일하게 늘림
        • flex-start : 위쪽 정렬
        • flex-end : 아래쪽 정렬
        • center : 가운데 정렬
        • baseline : 텍스트 기준선에 맞춘다
      • flex-wrap : 아이템이 컨테이너를 초과할 경우 줄바꿈 여부를 설정
        • nowrap : 기본값. 줄바꿈 불가
        • wrap : 줄바꿈 허용
        • wrap-reverse : 줄바꿈을 역방향으로 허용
    • 아이템 속성
      • flex-grow : flex 아이템이 남은 공간을 차지하는 비율
      • flex-shrink : flex 아이템이 줄어드는 비율
      • flex-basis : 아이템의 기본 크기
      • align-self : 특정 아이템의 교차 축 정렬
      • flex : 아이템 공간 분배를 설정

Grid

  • CSS Grid Layout은 웹 페이지에서 2차원 레이아웃(행과 열)을 설계하기 위한 강력한 도구
  • Flex와 달리 Grid는 행과 열을 동시에 제어할 수 있어 복잡한 레이아웃에도 적합

Grid 컨테이너와 Grid 아이템

  • Grid 컨테이너 : Grid 레이아웃이 적용된 부모 요소
  • Grid 아이템 : Grid 컨테이너 안에 포함된 자식 요소
  • 위치와 크기를 설정

컨테이너 속성

  • display : grid; Grid 레이아웃을 활성화
  • grid-template-rows : 행의 크기 정의
  • grid-template-columns : 열의 크기
    • 각 크기는 px, %, fr, auto 등을 사용
  • gap : 행과 열 사이의 간격 설정

아이템 속성

  • grid-column : 아이템이 차지할 열의 범위
  • grid-row : 아이템이 차지할 행의 범위
  • justify-self : 특정 아이템의 수평 정렬
  • align-self : 특정 아이템의 수직 정렬

정렬 속성

  • justify-items : Grid 아이템의 수평 정렬
  • align-items : Grid 아이템의 수직 정렬

flex와 grid의 간단한 차이점

  • flex는 아이템이 우선적으로 크기를 결정
  • grid는 그리드 템플릿 설정이 우선 적용

미디어 쿼리

  • 디바이스의 화면크기, 해상도, 방향에 따라 다른 스타일을 적용할 수 있도록 도와준다
  • 반응형 웹 디자인을 구현할 수 있다

미디어 쿼리 문법

  • max-width
    • 화면의 최대 너비를 기준으로 스타일 적용
    • @media (max-width: 768px) 화면 너비가 768px 이하일 때
  • min-width
    • 화면의 최소 너비를 기준으로 스타일 적용
    • @media (min-width: 1024px) 화면 너비가 1024px 이상일 때
  • min-width and max-width
    • 화면 너비 기준을 내가 정한 기준으로 적용
    • @media (min-width: 500px) and (max-width: 700px)
  • orientation
    • 디바이스의 방향을 기준으로 스타일 적용
    • landscape : 가로 모드
    • portrait : 세로 모드
    • @media (orientation: portrait) 세로 방향일 때
  • resolution
    • 화면 해상도(DPI)를 기준으로 스타일 적용
    • @media (min-resolution: 300dpi)
